Hiện tại token chỉ là số nguyên — model chưa thấy mối quan hệ nào giữa chúng.

2 Embedding lookup (1, 4) → (1, 4, 128)

Mỗi token ID lookup ra vector 128 chiều (học được). Nhân thêm √d_model = √128 ≈ 11.3 theo paper.

Embedding ngẫu nhiên ban đầu, sẽ học qua training. Mỗi ô màu = 1 chiều của vector 128-d.

3 + Positional Encoding (1, 4, 128)

Cộng vào embedding một tensor sin/cos cố định theo vị trí. Cho phép model phân biệt token ở pos 0 vs pos 3 (vì self-attention bản chất là permutation-invariant).

Tensor sau cộng vẫn shape (1, 4, 128) — vào encoder layer tiếp theo.

8 Decoder Masked Self-Attention (1, 1, 128) → (1, 1, 128)

Decoder cũng có self-attention nhưng causal mask: token tại pos i chỉ được attend đến pos ≤ i. Ngăn "nhìn trộm" tương lai khi train.

Causal mask khi tgt có 4 token
sáng = nhìn được, tối = bị mask
Khi sinh token thứ 2, decoder mới thấy <sos>.
Khi sinh token thứ 3, decoder thấy <sos> + token vừa sinh.
Cứ thế tiếp.

9 Decoder Cross-Attention Q ← decoder, K,V ← memory

Đây là chỗ decoder "đọc" lại src. Q từ decoder hidden state, K và V từ memory (output encoder). Cho phép mỗi token tgt attend đến bất kỳ token src nào.

12 Tóm tắt

Encoder làm gì

  • Embed + position
  • Self-attention: token "đọc" token khác trong src
  • FFN: biến đổi từng vị trí
  • Lặp lại × N layer → memory

Decoder làm gì

  • Masked self-attention: nhìn token đã sinh
  • Cross-attention: nhìn memory (chỗ học reverse)
  • FFN
  • Lặp × N layer, rồi Linear → vocab
  • Argmax, append, lặp đến EOS
